# ruyi

内部实现了一个日志线程专门打印网络事件，这是典型的SPSC模式，在现代SPSC的设计中，（双缓冲/三缓冲/环形数组）结合无锁方案能达到最大性能，这里选择了环形数组，基于以下两点考虑：

1. 把日志时间交给日志线程获取，尽可能的减少网络线程压力
2. 尽可能减少日志处理粒度，防止崩溃时损失过多日志

对于第一个理由，这里认为对于网络事件，损失一点时间上的精度是可以接受的，只要确保日志顺序正确即可，还可以一定程度上减少网络线程的压力，这是采用缓冲策略做不到的；对于第二个理由，这是双缓冲
本身固有的缺陷，双缓冲只有在缓冲区满时或者到达一定等待超时时间才会提交buff，缓冲区满时提交会导致日志处理粒度增大，到达一定空闲时间提交又会使其失去原本意义

可靠性

但是需要明确，采用上面的方案，写入空间满时会出现日志丢失的情况，这是不可避免的，采用消息队列的设计能避免这种情况；由于日志线程只打印连接，出错信息，正常情况下，一个不需要很大的缓冲区就足以处理所有日志，如果出现日志丢失的情况，可以考虑是出现了其它BUG
